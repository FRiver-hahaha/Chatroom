# 前言

设计一个大型的项目需要好的架构，我将在本md文件中，逐步建立Chatroom的架构。

[官网要求](https://plan.xiyoulinux.com/plan/plan7-chatroom)

## 整体结构分层

1. 界面层
2. 业务层
3. 传输层
4. 开发层

## 界面内容

采用对应语言的UI库，基于下层模块设计上层UI

## 业务模块分割

1-4参考[官网要求](https://plan.xiyoulinux.com/plan/plan7-chatroom)

1. 帐号模块 -> 依赖：无
2. 好友模块 -> 依赖：账号模块
3. 群组模块 -> 依赖：账号模块，部分好友模块
4. 聊天模块 -> 依赖：账号模块，好友模块，群组模块
5. 管理员模块：服务器日志 -> 依赖：无
6. 面向用户模块：分为普通用户和自定义用户(可以自行设定IP:PORT) -> 依赖：无
7. 动态模块：私聊和群组小游戏，朋友圈动态，聊天持续天数 -> 依赖：账号模块，好友模块，群组模块

## 传输模块分割

1. 服务器模型：主从Reactor模型
2. 存储方式：Mysql + Redis冷热搭配，序列化和反序列化
3. 并发的优化：线程池，协程
4. 加密：TLS
5. TCP心跳检测

## 开发模块分割

1. 开发语言：C++ + Protobuf

2. 数据结构：  
    元数据：用户信息，好友关系，群组状态(读多写少) -> 邻接表 + 红黑树，节省空间 + 快速读取
    消息数据：聊天内容(写密集，时序性强) -> RingBuffer + Redis List
    文件数据：图片，视频，文件(一次写入，多次读取) -> 单独设计
    状态数据：好友在线状态，会话(变化频繁) -> 推拉结合模型：Redis (TimeLine)List

3. 构建与测试：CMake
